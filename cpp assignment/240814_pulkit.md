# Memory-Efficient Versioned File Indexer

## Overview

This project implements a **memory-efficient log file analyzer** in C++.  
The program processes large log files using **fixed-size buffered reading** and supports multiple query types for analyzing word frequencies.

The analyzer is designed to efficiently handle large datasets (100MB+) while maintaining a **constant memory footprint** and demonstrating key **Object-Oriented Programming concepts** required by the assignment.

## Program Architecture

The program follows an **Object-Oriented Design** consisting of the following main classes:

### 1. `BufferedReader`
Handles efficient file reading using fixed-size buffers.

Responsibilities:
- Opens files using `fopen`
- Reads file chunks using `fread`
- Ensures constant memory usage

---

### 2. `Tokenizer`
Responsible for converting raw text into words.

Features:
- Extracts **alphanumeric tokens**
- Converts characters to lowercase for case-insensitive comparison
- Handles **words split across buffer boundaries**

---

### 3. `VersionedIndexer`
Maintains a frequency index of words for a file version.

Responsibilities:
- Builds a word-frequency map
- Supports querying word counts
- Returns top-K most frequent words

---

### 4. `QueryProcessor` (Abstract Base Class)

Defines the interface for processing queries.

Derived classes:
- `SingleVersionQuery` – handles **word** and **top** queries
- `DiffQuery` – handles **difference** queries across two files

This demonstrates **inheritance and runtime polymorphism**.

## Performance Optimizations

The program is optimized for processing large files:

### 1. Buffered File Reading
Files are read using `fread` with configurable buffer size (256–1024 KB).

This reduces system calls and improves disk throughput.

---

### 2. Streaming Tokenization
The tokenizer processes text **directly from buffers**, avoiding unnecessary string allocations.

---

### 3. Hash-Based Frequency Counting
Uses `unordered_map` for near **O(1)** average insertion and lookup.

---

### 4. Preallocated Hash Table
The frequency map uses:
freq.reserve(...)


to reduce rehashing overhead.

---

### 5. Minimal Memory Usage
The program only stores **unique words**, not the entire file.

This ensures scalability for very large logs.

---

## Assumptions

- Words are defined as **contiguous alphanumeric sequences**.
- Matching is **case-insensitive**.
- Buffer size must be between **256 KB and 1024 KB**.

---

## Example Workflow

1. The program reads file chunks using `BufferedReader`.
2. `Tokenizer` extracts words from each chunk.
3. `VersionedIndexer` updates frequency counts.
4. `QueryProcessor` executes the requested query and prints results.
